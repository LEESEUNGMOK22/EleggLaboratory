# 법적/운영 사용 가이드 (프로젝트 내부용)

마지막 확인: 2026-03-09 (KST)

## 이 프로젝트에서 할 수 있는 것
- SRD 범위의 규칙 구조 사용
  - 6능력치, d20 판정, 숙련 보너스, 이점/불리점, AC/HP/세이브 같은 메커니즘
- SRD 공개 범위의 수치/절차를 데이터화
- "5E 호환 감각"의 디지털 자동 진행 UX로 재해석
- 클래스 전투 성향, 판정 로직, 이벤트 분기 로직을 오리지널 구현으로 설계

## 피해야 하는 것
- D&D 상표/로고/브랜드 자산 의존
  - 게임 타이틀/메인 브랜딩에 D&D 표기를 핵심 식별자로 사용 금지
  - 공식 로고, 앰퍼샌드 마크, 공식 UI 트레이드드레스 모사 금지
- 공식 세계관/고유명사/상징적 인물·지명·진영 복제
- 공식 일러스트/토큰/룰북 문장 대량 복사
- "공식 제품"처럼 오인될 수 있는 표기

## 오리지널 명칭으로 바꿔야 하는 영역
- 도시, 국가, 길드, 교단, 귀족가, 유적, 역사 사건
- 몬스터 명칭(고유 설정 의존 명칭은 일반 판타지 원형으로 재명명)
- 퀘스트 제목, 유물 이름, 세력명
- UI 아이콘/문구/레이아웃의 브랜드 시그니처 요소

## 게임 내 표기 원칙 (5E 감각 유지 + 브랜드 비의존)
- 허용: "d20 기반 판정", "숙련 보너스", "이점/불리점" 같은 일반 규칙 용어
- 지양: "D&D 공식", "Forgotten Realms" 등 특정 IP 연상 문구
- 권장: "SRD 호환 구조", "고전 판타지 TRPG 감각" 같은 중립 표현

## Attribution/NOTICE 운영 메모
- SRD 공개본의 정확한 attribution 및 라이선스 고지는 반드시 원문 확인 후 반영한다.
- 원문을 여러 파일에 중복 복붙하지 않고, 최종 배포 시 단일 NOTICE/ATTRIBUTION 파일로 관리한다.

## 법적 해석 범위 주의
- 이 문서는 법률 자문이 아니라 프로젝트 운영 가이드다.
- 애매한 항목은 항상 보수적으로 판단하고, 필요 시 전문 자문을 거친다.

## 이 문서가 실제 게임 기획/UX에 어떻게 쓰이는지
- 기획자는 세계관 문안/명명 작업 전에 본 문서의 "금지/치환" 목록을 먼저 확인한다.
- UI/마케팅 카피 작성자는 "오인 방지" 원칙을 QA 체크리스트에 넣는다.
