# Rules Core (자동 진행 RPG 번역판)

기준: SRD 5.1 우선, 5.2.1 비교 보조
마지막 확인: 2026-03-09 (KST)

## 1) 능력치 6종
- STR/DEX/CON/INT/WIS/CHA
- 자동 진행 번역: 모든 자동 행동은 "행동 타입별 주능력치"를 참조

## 2) 수정치(Modifier)
- 능력치에서 파생되는 정수 보정값
- 자동 진행 번역: AI 전투 선택/탐험 성공률/사회 판정 모두 수정치 기반 확률로 통일

## 3) d20 판정 감각
- 기본: d20 + 능력 수정치 + (해당 시) 숙련 보너스
- 자동 진행 번역: 플레이어 입력 없이 백엔드에서 굴리고, 로그에 결과만 표시

## 4) 숙련 보너스
- 레벨 기반 공통 보너스
- 자동 진행 번역: 숙련 보너스는 클래스 정체성보다 "성장 체감"의 핵심 수치

## 5) 난이도(DC) / 이점 / 불리점
- DC 단계(쉬움~매우 어려움)
- 이점/불리점은 2회 굴림 중 높은 값/낮은 값
- 자동 진행 번역: 이벤트 메타에 `advantageState`를 붙여 서사 선택이 확률에 반영되게 함

## 6) 패시브 판정 감각
- 수동 굴림 대신 상시 감지값으로 처리 가능한 구조
- 자동 진행 번역: 함정 감지, 매복 탐지, 거짓말 간파를 패시브 지표로 주기적 평가

## 7) 세이브 개념
- 특정 위험(독, 공포, 주문, 환경) 저항 판정
- 자동 진행 번역: 조건 이상 부여/해제 시 세이브 파이프라인을 공통 함수화

## 8) 레벨 성장
- 레벨업 시 내구/성능/자원 관리 능력 확대
- 자동 진행 번역: 수동 스킬트리 대신 "클래스 성향 파라미터"와 "자동 행동 우선순위"가 함께 성장

## 최소 구현 공식(추천)
- `checkTotal = d20 + abilityMod + proficiency + situationalMod`
- `success = checkTotal >= DC`
- `adv/dis`: 주사위 단계에서만 처리

## 이 문서가 실제 게임 기획/UX에 어떻게 쓰이는지
- 시스템 팀이 공통 판정 엔진 설계 시 단일 기준으로 사용
- UX 팀이 로그/요약 UI에서 어떤 수치를 보여줄지 결정할 때 사용
