# Rules Combat (자동 전투 번역 규칙)

기준: SRD 5.1 우선
마지막 확인: 2026-03-09 (KST)

## 핵심 전투 구성
- 명중 판정: d20 + 관련 보정 vs AC
- 피해 감각: 무기/주문 기반 피해 주사위 + 능력/기타 보정
- 이니셔티브: 전투 시작 순서 결정(DEX 감각)

## 자동 진행형 단순화 원칙
- 턴별 수동 입력 제거
- 행동/보너스 행동/반응은 AI 스케줄러가 자동 배치
- 플레이어는 전투 중 조작 대신 "운영 원칙(공격적/안전/보존)"만 설정

## 단순화해도 남겨야 할 D&D풍 감각
- 명중 vs AC의 이분 구조
- 치명타/실패 체감
- 클래스별 행동 우선순위 차이
- 상태이상과 세이브의 긴장감

## 상태 이상(Conditions) 처리
- 예: 기절, 속박, 넘어짐, 독, 공포, 실명 등
- 자동 진행 번역:
  - 상태 부여 시 즉시 태그화
  - 턴 시작/종료 세이브로 해제 판정
  - 로그는 요약/상세 2단계 표시

## 근접/원거리/주문 전투 차이
- 근접: 위치/접촉/생존 부담
- 원거리: 거리/엄폐/안정 화력
- 주문: 자원 슬롯/집중/광역 제어
- 자동 진행 번역: 전투 AI가 거리, 위험도, 자원 상태로 자동 선택

## 휴식과 자원 회복
- 짧은 휴식/긴 휴식 감각을 유지
- 자동 진행 번역:
  - 일반 루프에서는 자동 휴식
  - 장기 자원 회복 타이밍은 이벤트/캠프 상태에 연동

## 구현 최소 기준(권장)
- 전투 요약 로그 기본 + 라운드 상세 로그 옵션
- 전투 종료 후 피로/상처/자원 감소를 탐험 루프로 전달

## 이 문서가 실제 게임 기획/UX에 어떻게 쓰이는지
- 전투 엔진 난이도/속도 밸런스 기준
- "전투는 자동" 원칙을 유지하면서도 클래스 판타지를 살리는 QA 기준
