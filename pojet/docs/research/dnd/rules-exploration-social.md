# Rules Exploration & Social (탐험/사회 축)

기준: SRD 5.1 우선
마지막 확인: 2026-03-09 (KST)

## 3축 원칙
- 이 프로젝트는 전투만이 아니라 탐험/사회/전투 3축이 모두 있어야 함

## 탐험 규칙 요소
- 지각(Perception), 조사(Investigation), 은신(Stealth), 생존(Survival)
- 추적, 함정 감지/해제, 여행 위험, 환경 피해, 야영/휴식
- 자동 진행 번역:
  - 지역마다 탐험 이벤트 풀을 두고 패시브+능력 판정으로 자동 해결
  - 중요한 위험만 T2/T3 선택 이벤트로 승격

## 사회 상호작용 요소
- 설득, 기만, 협박, 통찰
- 평판, 소문, 세력 호감/적대, 동료 관계 변화
- 자동 진행 번역:
  - 사소한 관계 변화는 T1 자동 처리
  - 세력 가입/배신/관계 확정은 T3 필수 정지

## 자동 진행 로그형 변환 제안
- 한 문장 로그: 일상 판정 결과
- 이벤트 카드(2~3문단): 맥락/위험/보상
- 선택지 3개 내외: 태도/윤리/야망 분기
- 미응답 처리 규칙: T2는 성향 프리셋 기반 자동

## 추천 데이터 포인트
- `explorationRisk`, `socialHeat`, `reputationDelta`, `relationshipDelta`, `futureFlag`
- 판정 성공/실패가 다음 지역 이벤트 가중치에 누적되도록 설계

## 이 문서가 실제 게임 기획/UX에 어떻게 쓰이는지
- 퀘스트/이벤트 파이프라인에서 T0~T3 승격 기준 정의
- 로그 밀도와 선택 빈도를 조절하는 UX 밸런스 기준
