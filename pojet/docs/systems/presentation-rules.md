# Presentation Rules

- 어떤 상황에서도 캐릭터 패널은 사라지지 않는다.
- 결정이 클수록(T3) 캐릭터 패널 강조를 키운다.
- 관계 변화는 표정/조명/거리감으로 체감시킨다.
- 장비 변경은 수치 외형 태그로 즉시 반영한다.
- 오염/축복은 미세하지만 누적되는 시각 언어를 가진다.
- 런 종료는 메뉴가 아니라 "의식"처럼 연출한다.
- 연대기는 로그가 아니라 문서/가계도 감각으로 읽게 한다.

## 이 문서가 실제 게임 기획/UX에 어떻게 쓰이는지
- UI 연출 우선순위 기준
- 과도한 효과를 억제하는 품질 기준
