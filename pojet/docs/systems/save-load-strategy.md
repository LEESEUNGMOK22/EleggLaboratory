# Save Load Strategy

## 저장 단위
- 런 스냅샷: 현재 진행 상태
- 연대기 아카이브: 완료 런 기록(요약 중심)

## 저장 시점
- 수동 저장 버튼
- 자동 저장: N틱(기본 6틱)마다
- 종료 직전 저장: run end 직전에 강제 기록

## 복원 시 재계산
- 파생 초상화 상태
- 패널 강조 상태
- 이벤트 타이머 잔여시간(보수적으로 초기화)

## 버전/마이그레이션
- `saveVersion` 포함
- 버전 불일치 시 기본 마이그레이션 + 누락 필드 보정

## 무결성/복구
- JSON 파싱 실패 시 최근 자동저장 fallback
- 둘 다 실패하면 새 런 시작 + 경고 로그

## 로그 저장 전략
- 전체 로그는 최근 N개(예: 240) 저장
- 연대기는 요약 문장 + 주요 선택 중심

## 이 문서가 실제 게임 기획/UX에 어떻게 쓰이는지
- 저장 정책과 성능/안정성 균형 기준
