# Content Style Guide

## 로그 문장 규칙
- 1문장 중심, 12~28자 권장, 동사로 끝나는 리듬 유지.

## 이벤트 본문 길이
- T2: 1~2문단(짧은 맥락+갈등)
- T3: 2문단(결정의 무게 강조)

## 선택지 문장 톤
- 명령형 대신 의지형: "받는다/거절한다/보류한다"
- 결과를 노골적으로 스포일러하지 않음.

## 고유명사 작명 규칙
- 도시/세력/인물 모두 오리지널 접두/접미 규칙 사용.
- 기존 TRPG 유명명과 음운 유사도 높은 명칭 회피.

## 오리지널 세계관 유지
- 규칙 구조는 호환 감각, 사건/지명/신앙은 독자 IP.

## D&D 느낌은 살리되 복제하지 않는 작법
- 수치/판정 감각만 차용, 문장과 설정은 완전 재창작.

## 반복 이벤트 변주법
- 동일 이벤트라도 주체/장소/감정 단어를 교체하는 템플릿 사용.

## 금지
- 지나치게 현대적 밈, 인터넷 슬랭, 메타 농담 남용 금지.

## 이 문서가 실제 게임 기획/UX에 어떻게 쓰이는지
- 신규 콘텐츠 PR 검수 기준
- 라이팅 팀 통일 스타일 가이드
