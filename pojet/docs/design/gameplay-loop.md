# Gameplay Loop

## 루프 요약
1. 런 시작/캐릭터 생성
2. 자동 틱 진행 (탐험/사회/전투/휴식)
3. 보상/피해/관계/평판 반영
4. 이벤트 디스패치(T0~T3)
5. T2는 제한시간 후 프리셋 자동 처리
6. T3는 강제 정지+플레이어 선택
7. 퀘스트/막 진행, 레벨업, 연대기 기록

## 틱 처리 우선순위
- phase resolve -> checks -> rewards -> event dispatch -> state transition -> ui sync

## 자동/수동 경계
- 자동: 이동, 일반 전투, 일반 루팅, 반복 의뢰
- 수동: 세력, 서브클래스, 유물, 계승, 관계 확정

## 이 문서가 실제 게임 기획/UX에 어떻게 쓰이는지
- 구현팀의 루프 컨트롤러 기본 순서
- QA의 자동/수동 경계 검증 기준
