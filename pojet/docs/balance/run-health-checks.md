# Run Health Checks

## 10분 체크
- 첫 10분 동안 동일 로그 패턴만 반복되지 않는다.
- 첫 의미 이벤트(T2)가 늦어도 12분 이내에 관측된다.
- 클래스별 최소 1회 강점 체감(전투/탐험/사회 중 1축 이상)이 있다.

## 20분 체크
- 첫 T3가 지나치게 늦지 않다(목표: 평균 8~30분).
- 관계 또는 세력 변화가 최소 1회 이상 명확히 기록된다.
- 퀘스트 갱신/레벨업 중 최소 하나가 발생한다.

## 시스템 일관성 체크
- 패널 강조 타이밍(T2/T3/레벨업)과 로그 기록 시점이 어긋나지 않는다.
- 세이브/로드 이후 이벤트 주기와 배속 체감이 유지된다.
- 자동화 프리셋 변경 후 T2 자동 처리 결과가 즉시 반영된다.

## 콘텐츠 분포 체크
- 이벤트 풀이 특정 클래스만 과도하게 편중되지 않는다.
- 관계/사회 카테고리가 전체 노출에서 하한(8%) 미만으로 떨어지지 않는다.
- 동일 문장 반복률이 허용치(중복 35%)를 넘지 않는다.

## 실패 시 우선 조치
- 스폰 확률 조정 -> 쿨다운 조정 -> 보상량 조정 순으로 적용.
- 이벤트 텍스트 중복은 변주 라인 추가로 우선 해결.
- 조기 사망 과다는 HP/회복보다 피로/공급 소모 곡선부터 완화.

## 이 문서가 실제 게임 기획/UX에 어떻게 쓰이는지
- 개발 빌드의 플레이 가능 여부를 5분 내 판정하기 위한 체크리스트.
