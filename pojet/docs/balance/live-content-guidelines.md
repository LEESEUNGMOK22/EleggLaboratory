# Live Content Guidelines

## 이벤트 티어 운영 비율
- 신규 추가분 기준 권장 비율: T0 50 / T1 28 / T2 16 / T3 6
- 월간 누적에서 T2/T3 비율이 올라가면 T0/T1 보강을 먼저 수행한다.

## 지역 추가 최소 묶음
- 지역 1개 추가 시 최소:
  - T0 12개
  - T1 8개
  - T2 4개
  - T3 2개
  - 로그 라인 20개
  - 일상 의뢰 5개

## 세력 추가 최소 연결점
- 세력 평판 보상 3단계(호의/중립/적대)
- 세력 관련 T2 2개, T3 1개
- 기존 세력 2개 이상과 갈등/연계 훅
- NPC 2명 이상, 동료 또는 경쟁자 1명 이상

## 동료 추가 최소 연결점
- 전투 역할 + class AI 태그
- 관계 이벤트 3개 이상(T1/T2/T3 혼합)
- 퀘스트 훅 2개 이상
- 초상화 상태 태그 4개 이상 연동

## 신규 클래스/서브클래스 추가 체크
- class-ai-presets 업데이트
- decision 추천 바이어스 정의
- 보상 테이블의 장비/자원 편향 점검
- 이벤트 classAffinity 훅 최소 6개 연결

## 반복 이벤트 변주 규칙
- 동일 이벤트는 최소 3개 문장 변주 세트 유지
- 고정 문구는 핵심 어절만 공유하고 동사/결말은 분기
- 중복 감지 경고(35% 초과) 시 우선 보강 대상 지정

## 과잉 팝업 방지 기준
- 평균 T2 간격 8틱 미만이면 즉시 경고
- 평균 T3 간격 18틱 미만이면 강제 감산
- 동일 카테고리 팝업 연속 2회 초과 금지

## 이 문서가 실제 게임 기획/UX에 어떻게 쓰이는지
- 라이브 업데이트 시 자동 진행 리듬 붕괴를 막는 배포 가이드.
