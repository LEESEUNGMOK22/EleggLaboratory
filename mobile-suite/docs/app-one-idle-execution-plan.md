# App One 실행 플랜 (Idle Merge + Clicker)

이 문서는 전달받은 기획 초안을 바탕으로, 실제 개발 가능한 형태로 재정리한 실행 계획이다.
핵심 방향: **광고/수익화는 후순위**, 먼저 코어 게임성 완성.

## 0. 제품 정의 (요약)
- 장르: Merge + Idle + Clicker
- 한 줄: 랜덤 기본원소를 소환/머지하고, 시간 변성으로 보상이 생기며, 오프라인 동안 성장하는 원소 숙성 게임
- 차별점:
  1) 시간 변성이 전략이 되는 구조
  2) 오프라인 보상 + 소환권으로 재접속 동기 강화
  3) 로그북으로 “방치 중 사건”을 플레이 재미로 전환

---

## 1. 개발 원칙
1. **MVP 우선**: 5x6 보드 + 4계열/4단계 + 티어 1~8 + 로그 요약
2. **데이터 주도**: 원소/변성/업그레이드를 테이블화
3. **오프라인 정확도**: elapsed 기반 정산(소환권, 생산, 변성)
4. **로그 압축**: 이벤트 폭주 방지(그룹 집계)
5. **광고 후순위**: 코어 루프가 재미 검증되기 전 BM 연동 금지

---

---

## 3. 완료 기준 (MVP)
- 10분 플레이에서 루프가 자연스럽게 순환
- 오프라인 복귀 시 정산이 직관적이고 유용
- 로그북이 “왜 성장했는지”를 설명
- 강제 광고 없이도 리텐션 지표 측정 가능

---

## 4. 리스크 및 대응
- 리스크: 이벤트 로그 과다
  - 대응: (from,to,tier) 그룹 요약 기록
- 리스크: 변성/생산 동시 계산 복잡도 증가
  - 대응: 정산 엔진 분리, 단위테스트 우선
- 리스크: 숫자 밸런싱 불안정
  - 대응: 테이블 기반 + 프리셋 난이도

---

## 5. 현재 결정 사항
- 광고 BM: **최후순위**
- 우선순위: 코어 루프 → 오프라인 → 로그북 → 업그레이드
- 방향성: 초기 기획은 존중하되, 플레이 체감에 맞춰 자유 첨삭 허용
